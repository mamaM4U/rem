# Arisu Landing Page

The newest addition to the Rem family, serving as the welcoming face of the workspace.
Built with Jaspr, Arisu presents the Rem to the world with elegance and grace, showcasing the harmony between Arona and Plana.

> **Note**: For complete development guide and workspace setup, see the [main README](../README.md) at the root of the repository.

## Quick Start

### Using Melos (Recommended)

From the workspace root:

```bash
# Start development server
melos arisu

# Build for production
melos arisu:build
```

### Running Directly

```bash
# From arisu directory
dart pub get
jaspr serve
```

Development server will be available at: http://localhost:8080

## Architecture

### Framework
- **Jaspr** - Dart web framework with server-side rendering

### Key Directories
- `lib/pages/` - Page components (Home, Blog, About)
- `lib/components/` - Reusable UI components (Header)
- `lib/constants/` - Theme colors and environment config
- `lib/models/` - Data models for API responses
- `lib/services/` - API service for fetching data from Plana
- `web/` - Static assets and entry point

### Pages
- **Home** (`/`) - Landing page showcasing Rem, Arona, Plana, and Arisu
- **Blog** (`/blog`) - Blog posts fetched from Plana API
- **About** (`/about`) - Detailed information about the workspace

## Development

### Hot Reload

Jaspr supports hot reload out of the box:
```bash
jaspr serve
```

### Build for Production

```bash
# Build static site
jaspr build

```

The project uses `jaspr_lints` for Jaspr-specific linting rules:
```bash
dart analyze
```

## Styling

Arisu uses Jaspr's CSS-in-Dart system for type-safe styling:

```dart
@css
static List<StyleRule> get styles => [
  css('.card', [
    css('&').styles(
      padding: Padding.all(1.rem),
      backgroundColor: cardBg,
      radius: BorderRadius.circular(8.px),
    ),
  ]),
];
```

### Theme Colors

Defined in `lib/constants/theme.dart`:
- **Primary** - Indigo (#6366f1)
- **Arona** - Blue (#3b82f6)
- **Plana** - Violet (#8b5cf6)

- **Framework**: Jaspr 0.21.7

## Docker

Arisu can be run in a Docker container, either standalone or as part of the full Rem stack.

### Build Docker Image

```bash
# From landing_page_ssr directory
docker build -t arisu .

# Run standalone
docker run -p 3000:8080 arisu
```

### Full Stack with Docker Compose

From the workspace root, run all services together:

```bash
# Start all services (DB + Plana + Arisu)
melos docker:up

# Or directly with docker-compose
docker-compose up -d
```

Services will be available at:
- **Arisu (Landing Page)**: http://localhost:3000
- **Plana (API Server)**: http://localhost:8080
- **PostgreSQL**: localhost:5432

### Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `API_URL` | Plana API server URL | `http://localhost:8080` |

In Docker Compose, Arisu connects to Plana via internal network: `http://plana:8080`

## API Integration

Arisu fetches data from Plana API server using server-side rendering:

```dart
// lib/services/api_service.dart
class ApiService {
  Future<ApiResponse<HomeData>> getHomeData() async {
    final response = await http.get(Uri.parse('$baseUrl/home'));
    // ...
  }
}
```

The Blog page uses `PreloadStateMixin` to fetch data during SSR and `SyncStateMixin` to sync state to the client:

```dart
class BlogState extends State<Blog> with PreloadStateMixin, SyncStateMixin<Blog, Map<String, dynamic>> {
  @override
  Future<void> preloadState() async {
    // Fetch data on server during pre-rendering
    final response = await ApiService().getHomeData();
    items = response.data!.items;
  }
}
```

```bash
cp .envied.example .envied
```

2. Run build_runner to generate env code:
```bash
dart run build_runner build
```
